All right, last thing we will demonstrate is how we can use relative state navigation to go UP the state tree. You can of course transition to a state if you know the full name (component1.component2.component3 for example) but You can also traverse up states kind of like a file tree system. If an ancestor exists, you can jump to the parent state, grandparent, great grandparent etc. Each of these components have been made a child state of the previous one so play with these buttons to get a feel for how the relative navigation works.

  • Go to parent state ('^')
  • Go to grandparent state ('^.^')
  • Go to great grandparent state ('^.^.^')
  • Go to grandparent state and then a child state called ".component4" ('^.^.component4')
